**[상황 설명]**
현재 Next.js 기반 웹앱에서 기존에 개별 컴포넌트 단위로 설정된 반응형 브레이크포인트(`@media (max-width: 1800px)`)들을 모두 걷어내고, 화면 해상도에 맞춰 UI가 비율에 맞게 줄어드는 **스케일 축소(Proportional Scaling) 방식**으로 일괄 전환하려고 해.
하지만 단순하게 16:9 컨테이너 통째로 화면 중앙에 스케일링을 걸어버리면, 맥북(16:10) 같은 환경에서는 위아래나 좌우로 레터박스(공백)가 크게 생기는 문제가 있어. 

그래서 전체를 통째로 묶는 게 아니라, **상단 네비게이션과 좌측 사이드바는 원래 레이아웃 영역을 유지하되 내부에만 스케일을 적용하고, 가장 중요한 중앙/하단 메인 컨텐츠 영역(페이지 영역)은 16:10이든 16:9든 가급적 화면에 꽉 차게 유연한 스케일링을 적용**하고 싶어.

이 방식은 추후 세부 반응형 CSS를 다시 짤 때 걷어낼 예정이므로, 나중에 지우기 쉽도록 롤백을 고려한 코드로 작성해줘.

**[작업 요구 사항]**

**1. 기존 임시 브레이크포인트 일괄 해제 (주석 처리)**
- `src/app/page.tsx`, `feature-section.tsx`, `simulation-table.tsx` 등에 퍼져있는 `@media (max-width: 1800px) { ... }` 코드를 삭제하지 말고 명시적으로 주석 처리해줘.
- 예: `/* [TEMP_SCALE_MODE_DISABLE] 차후 반응형 작업 시 복구 */`

**2. 영역별(Area-based) 분리 스케일링 래퍼 설계 (가장 핵심)**
- 전체 화면을 하나의 큰 판으로 묶어서 줄이는 게 아니라, `AppLayout`(또는 최상단 커스텀 레이아웃) 구조를 유지한 상태에서 영역별로 스케일 로직을 분리해줘.
   - **사이드바(Sidebar) / 상단 네비(Header):** 
     자신들의 고정된 Layout 공간(예: 세로 100vh, 좌측 고정너비)은 유지하되, 내부의 아이콘이나 텍스트 요소들만 브라우저 비율에 따라 적당한 `transform: scale()`로 줄어들게 해줘. 공간 자체가 줄어들어 레이아웃이 깨지지 않게 `transform-origin: top left` 등을 사용해.
   - **메인 컨텐츠 영역 (하단/중앙 페이지부):**
     16:9 화면이나 맥북의 16:10 등 어떠한 비율에서도 상하좌우 공백(레터박스)이 최소화되고 화면에 꽉 차 보이도록, `width: 100%`, `height: 100%` 컨테이너 내부에서 컨텐츠 비율을 조율해줘. 필요하다면 `Math.max(비율)`을 쓰거나, 요소 간 간격(`gap`) 혹은 플렉스 구조를 조절해서 **비율 왜곡(찌그러짐) 없이 꽉 채우는 유연한 Scale 로직**을 만들어줘.

**3. 화면 왜곡 및 요소 찌그러짐 원천 차단**
- 세로가 긴 해상도(맥북 16:10)나 가로가 긴 해상도(TV 16:9)로 줄어들거나 늘어나도, **차트 컨테이너나 이미지가 가로/세로 비율이 무너져 평면적으로 납작해지는(aspect-ratio 붕괴) 현상이 절대 없어야 해.**
- 스케일링 중 여백이 남더라도 컨텐츠 자체가 틀어지지 않도록, 여백을 마진(Margin)이나 `justify-content` 류의 정렬로 부드럽게 흡수하도록 부모 CSS를 조율해줘.

**4. 롤백을 위한 명시적인 컨벤션과 코드 분리 (필수)**
- 나중에 스케일 방식 자체를 걷어낼 예정이므로, 이번 작업으로 추가되는 Custom Hook(예: `useResponsiveScale`)이나 Wrapper 컴포넌트는 되도록 한두 개의 파일로 응집시키고 주석을 철저히 달아줘.
- 레이아웃 파일이나 기존 파일에 추가되는 코드는 앞뒤로 `// --- [TEMP_SCALE_START] 임시 스케일링 모듈 ---` 및 `// --- [TEMP_SCALE_END] ---` 처럼 식별자를 달아, 차후 이 주석만 찾아 지우면 원복될 수 있게 작업해줘.
